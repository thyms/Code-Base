Ubuntu 11.04 Installation After-works
-------------------------------------
--> Choose "Bridged Adaptor" from Settings->Network

--> VirtualBox Guest Utils
$sudo apt-get update
$sudo apt-get install virtualbox-ose-guest-utils


--> Windows 95-ish theme problem
-> Solution 1, requires every time rebooted.
$killall -9 gnome-settings-daemon && gnome-settings-daemon // For windows 95-ish theme fix
-> Solution 2, permanent
$gksudo gedit /etc/xdg/autostart/gnome-settings-daemon.desktop
- Change the line:
Exec=/usr/lib/gnome-settings-daemon/gnome-settings-daemon
- with this:
Exec=bash -c "sleep 15; /usr/lib/gnome-settings-daemon/gnome-settings-daemon"


--> Shared Folder, Mounting
-> You have to add "shared" folder from VirtualBox->Devices->Shared Folder
$sudo mkdir ~/Shared
$sudo mount.vboxsf shared ~/Shared
-> To make it mounted after every reboot
$sudo gedit /etc/rc.local
- Add this line:
mount.vboxsf shared ~/Shared

Installing Oracle Express Edition
---------------------------------
$sudo vi /etc/apt/sources.list
append this: deb http://oss.oracle.com/debian unstable main non-free
$wget http://oss.oracle.com/el4/RPM-GPG-KEY-oracle -O- | sudo apt-key add -
$sudo apt-get update
$sudo aptitude apt-get install oracle-xe-universal
$sudo /etc/init.d/oracle-xe configure
home page: http://127.0.0.1:8080/apex

to start manually: $sudo /etc/init.d/oracle-xe start (this is not working unless logged in with oracle user, try restart)
to restart manually: $sudo /etc/init.d/oracle-xe restart
to stop  manually: $sudo /etc/init.d/oracle-xe stop
to configure: $sudo /etc/init.d/oracle-xe configure
to view status: $sudo /etc/init.d/oracle-xe status
to edit configuration: $sudo vi /etc/default/oracle-xe

Error: /usr/lib/oracle/xe/app/oracle/product/10.2.0/server/bin/nls_lang.sh:114:[[:not found
Solution: replace '[[' with '[' and ']]' with ']'

Error: Cannot find 'sqlplus' command
Solution: Add content of '/usr/lib/oracle/xe/app/oracle/product/10.2.0/server/bin/oracle_env.sh' to ~/.bashrc
Steps:
Copy contents by using: $sudo vi /usr/lib/oracle/xe/app/oracle/product/10.2.0/server/bin/oracle_env.sh
Paste them to ~/.bashrc by using: $sudo vi ~/.bashrc

Error: Cannot login as 'sys' and/or 'system'
Solution: Add your user to DBA group. 
Steps:
$sudo usermod -a -G dba deniz
$ sqlplus "/ as sysdba"
SQL> show user
USER is "SYS"
SQL> passw system
Changing password for system
New password:
Retype new password:
Password changed
SQL> quit

$ sqlplus system/password
SQL> passw sys

SQL> quit
